# 状态驱动自适应调度系统 — 技术方案

## 核心设计理念

> **事件只是状态变化的表现。船是唯一执行单元，重分配以船的状态为核心决策依据。任务和路网作为影响因子输入，不单独评分。**

```
传统 (事件驱动):            本方案 (状态驱动):
  故障 → 重分配              Ship 状态变化 → 独立判断 → 分级响应
  低电量 → 重分配            每艘船自主决策: 正常/释放部分/释放全部/竞拍
  新任务 → 重分配            Task+Road 作为影响因子, 不独立评分
```

---

## 1. Ship 状态（6维）— 唯一决策依据

```
ShipState_i ∈ R^6:

┌──────────────────────────────────────────────────────────────┐
│ [0] energy_score    = remaining / needed_for_remaining       │
│                      剩余80kWh, 需要40kWh → 2.0 (安全)       │
│                      剩余30kWh, 需要60kWh → 0.5 (紧急)       │
│                                                              │
│ [1] load_ratio      = current_load / max_payload             │
│                      > 0.9 → 已满载, 不能接新任务             │
│                                                              │
│ [2] health          = 0~1                                    │
│                      < 0.3 → 故障风险                        │
│                                                              │
│ [3] idle            = 0/1                                    │
│                      task_sequence为空 + 负载为0 → 空闲       │
│                                                              │
│ [4] position_score  = dist_to_target / total_expected_dist   │
│                      > 1.2 → 偏离预期航线                     │
│                                                              │
│ [5] eta_reliability = actual_elapsed / estimated_time        │
│                      > 1.2 → 延误, 下游任务可能超期           │
│                      (受路网拥堵影响, Road信息由此输入)        │
└──────────────────────────────────────────────────────────────┘
```

---

## 2. Task 和 Road 的作用 — 影响因子

```
Task 和 Road 不独立评分, 而是作为影响因子输入到 Ship 状态计算中。

Task.urgency → 影响 ship.energy_score 的分母 (紧急任务需要更多能源预留)
  urgency = remaining_dist / remaining_time
  例: 距离20km, 还剩30min → urgency=6.7 → 预留能源 = 基础能耗 × urgency_factor

Road.congestion → 影响 ship.eta_reliability
  拥堵导致实际用时 > 预估用时 → eta_reliability > 1.0
```

## 3. 重分配决策逻辑

```python
def evaluate_ship(ship, tasks, road_network, current_time):
    """
    单船状态评估 → 重分配决策
    每艘船独立判断, 不依赖全局平均
    """
    # ── 计算6维状态 ──
    state = {}
    
    # energy_score: 剩余/所需
    needed = 0
    for tid in ship.task_sequence:
        task = tasks.get(tid)
        if task:
            d = road_network.dist_matrix[ship.current_node, task.pickup_node] + \
                road_network.dist_matrix[task.pickup_node, task.delivery_node]
            needed += d / 1000.0 * ship.energy_per_km
            # 紧急任务加权
            if task.deadline < float('inf'):
                remaining_t = max(1, task.deadline - current_time)
                urgency = (d / ship.max_speed) / remaining_t
                needed *= (1.0 + min(urgency, 2.0))
    
    state['energy_score'] = ship.energy / max(1, needed)
    
    # load_ratio
    state['load_ratio'] = ship.load / max(ship.max_payload, 1)
    
    # health
    state['health'] = ship.health
    
    # idle
    state['idle'] = 1.0 if (not ship.task_sequence and ship.load == 0) else 0.0
    
    # position_score: 偏离预期
    if ship.current_task_id >= 0:
        task = tasks.get(ship.current_task_id)
        if task and ship.current_phase == 'navigate_to_pickup':
            expected = road_network.dist_matrix[ship.current_node, task.pickup_node]
            state['position_score'] = min(2.0, expected / max(1, ship.total_distance))
        else:
            state['position_score'] = 1.0
    else:
        state['position_score'] = 1.0
    
    # eta_reliability: 受路网拥堵影响
    # 拥堵节点 → 实际用时 > 预估
    congestion_factor = 1.0
    for nid in [ship.current_node]:
        node = road_network.nodes.get(nid)
        if node and hasattr(node, 'congestion'):
            congestion_factor = max(congestion_factor, 1.0 + node.congestion * 0.5)

    elapsed = ship.total_time
    estimated = sum(
        road_network.dist_matrix[ship.current_node, tasks[tid].pickup_node] / ship.max_speed
        for tid in ship.task_sequence if tid in tasks
    ) if ship.task_sequence else 1
    state['eta_reliability'] = (elapsed / max(1, estimated)) * congestion_factor

    # ── 决策 ──
    if state['energy_score'] < 0.5:
        return ("release_all", "能源不足以完成剩余任务", state)
    if state['health'] < 0.3:
        return ("release_all", "故障风险", state)
    
    if state['energy_score'] < 0.8:
        n = int(len(ship.task_sequence) * (1 - state['energy_score']))
        return ("release_some", f"能源紧张, 释放{n}个任务", state)
    if state['eta_reliability'] > 1.5:
        return ("release_some", "严重延误, 转移下游任务", state)
    
    if state['idle'] > 0 and state['health'] > 0.7:
        return ("take_tasks", "空闲且健康, 可接任务", state)
    
    return ("normal", "", state)
```

## 4. 调度主循环

```python
def scheduling_loop(self, dt=5.0):
    """每个周期: 评估每艘船 → 收集异常 → 统一处理"""
    
    self.advance_time(dt)
    
    # ── 评估所有船 ──
    decisions = {}
    for sid, ship in self.ships.items():
        action, reason, state = evaluate_ship(
            ship, self.tasks, self.rn, self.current_time)
        decisions[sid] = (action, reason, state)
    
    # ── 分类汇总 ──
    release_pool = []       # 需要释放的任务
    idle_ships = []         # 空闲可接任务的船
    normal_ships = []       # 正常的船
    
    for sid, (action, reason, state) in decisions.items():
        if action == "release_all":
            for tid in list(self.ships[sid].task_sequence):
                task = self.tasks.get(tid)
                if task and task.status not in ('completed',):
                    task.status = 'pending'
                    task.assigned_ship = -1
                    release_pool.append(tid)
            self.ships[sid].task_sequence.clear()
            self.ships[sid].load = 0
            
        elif action == "release_some":
            n = int(len(self.ships[sid].task_sequence) * (1 - state['energy_score']))
            for tid in self.ships[sid].task_sequence[-n:]:
                task = self.tasks.get(tid)
                if task and task.status == 'pending':
                    task.assigned_ship = -1
                    release_pool.append(tid)
                    self.ships[sid].task_sequence.remove(tid)
                    self.ships[sid].load -= task.payload
                    
        elif action == "take_tasks":
            idle_ships.append(sid)
            
        else:
            normal_ships.append(sid)
    
    # ── 统一重分配 ──
    if release_pool or idle_ships:
        # 固定正常船, 释放的任务 + 空闲船参与拍卖
        fixed_ships = set(normal_ships)
        self._auction_with_pool(release_pool, idle_ships, fixed_ships)
    
    # 可视化
    self.visualizer.update(self.ships, self.tasks, ...)
```

## 5. 系统架构

```
传感器数据 (GPS/BMS/任务系统)
        │
        ├──→ Road.congestion ──→ 影响 ship.eta_reliability
        ├──→ Task.urgency    ──→ 影响 ship.energy_score (所需能耗)
        │
        └──→ 更新 Ship 状态 (6维)
                  │
         ┌────────┼────────┐
         │        │        │
    release_all  release_some  take_tasks
         │        │        │
         └────────┼────────┘
                  │
          收集释放任务 + 空闲船
                  │
             Auction 重分配
```

## 6. 与传统事件驱动对比

| | 传统事件驱动 | 本方案 |
|------|-----------|------|
| 决策主体 | 全局规则 | **每艘船独立判断** |
| 阈值来源 | 硬编码 20% | **动态计算: 剩余/所需** |
| 重分配粒度 | 全有或全无 | **三级: release_all / release_some / take_tasks** |
| Task/Road作用 | 独立评分再平均 | **作为影响因子输入 Ship 状态** |
| 论文创新性 | 低 | **高: 船中心决策 + 动态阈值** |
