# 基于AIS导航势场的无人艇路径规划 — 方法说明

## 一、系统概述

本系统用 AIS（船舶自动识别系统）历史轨迹数据构建**导航势场**（Φ̄ ∈ [-1,1]），引导 A* 路径规划沿历史通航习惯行驶。

**两层架构：**
- **离线层**（Python）：从138个AIS轨迹XLS文件生成导航势场CSV
- **在线层**（ROS C++）：加载势场，运行航道引导A* + 路径平滑

**系统拓扑：**
```
ais_planner_node       → /ais_planned_path       (航道引导A*)
                        → /ais_traditional_path   (传统A* 参照)
ais_path_optimizer     → /ais_smoothed_path       (航道约束平滑)
map_server             → /map                     (占用栅格地图)
```

## 二、导航势场 Φ̄ 构建

### 2.1 整体流程（6步）

```
138个AIS XLS 文件
  │
  ├─ Step 1: 读取 → 经纬度 + 对地航向(COG 0-360°)
  │
  ├─ Step 2: 经纬度→世界坐标(米) + 栅格网格(78×152@0.5m)
  │
  ├─ Step 3: 全区域COG直方图 → 全局参考方向 θ_ref (如112°)
  │           每栅格局部COG直方图 → 局部方向场 θ_local(p) [诊断用]
  │
  ├─ Step 4: 方向加权核密度估计(KDE)
  │           weighted(p) = Σ cos(COG_i - θ_ref) × exp(-||p-p_i||²/2h²)
  │           abs_w(p)    = Σ |cos(COG_i - θ_ref)| × exp(-||p-p_i||²/2h²)
  │
  ├─ Step 5: 归一化
  │           Φ̄(p) = weighted(p) / (abs_w(p) + ε)   ∈ [-1, 1]
  │
  └─ Step 6: 保存CSV (78×152) + 梯度CSV (∇Φ̄)
```

### 2.2 关键公式

**方向权重（余弦连续权重替代硬二分类）：**

```
w_i = cos(COG_i - θ_ref)

+1  = 船舶航向与参考方向完全一致  (顺流/下游)
-1  = 船舶航向与参考方向完全相反  (逆流/上游)
 0  = 船舶横穿 (垂直方向)
```

**方向加权KDE：**

```
                    Σᵢ w_i · K(p, p_i)
Φ̄(p)  =  ──────────────────────────────────  ∈  [-1, 1]
               Σᵢ |w_i| · K(p, p_i) + ε

其中  K(p, p_i) = exp( -‖p - p_i‖² / (2h²) )
      h = 5.0m (高斯核带宽)
```

**物理语义：**

| Φ̄(p) | 含义 |
|-------|------|
| → +1 | 纯顺流航道 (COG与θ_ref一致) |
| → -1 | 纯逆流航道 (COG与θ_ref相反) |
| ≈ 0  | 双向混合水域 / 交汇口 / 无历史交通 |

### 2.3 坐标系统

```
AIS 经纬度 (lon, lat)
    │  pixel_col = 1 + (lon-120.4147)/0.23 × 151
    │  pixel_row = 1 + (lat-32.0820)/(-0.1017) × 77
    ▼
世界坐标 (x, y 米) = (pixel_col×0.5, pixel_row×0.5)
    │  范围: X∈[0,76]m, Y∈[0,39]m
    ▼
栅格: 78行 × 152列 @ 0.5m/cell = 76m × 39m
    │  CSV保存: np.flipud 转换为图像约定 (行0=北)
    ▼
C++ loadCSVToLayer: transpose + reverse×2 → GridMap
    │  与PGM占用栅格地图对齐
    ▼
GridMap "phibar" 层 (152×78, 值域[-1,1])
```

## 三、A* 航道引导代价函数

### 3.1 航道合规代价

```
c_lane(p; d) = max(0, -d · Φ̄(p))
```

其中 `d ∈ {+1, -1}` 是期望航向（+1=顺流/下游，-1=逆流/上游）。

**行为：**

| 条件 | c_lane | 效果 |
|------|--------|------|
| d 与 Φ̄ 同号 (正确航道) | 0 | 无惩罚，自由通行 |
| d 与 Φ̄ 异号 (错误航道) | \|Φ̄\| | 按势场偏差正比惩罚 |
| Φ̄ ≈ 0 (混合/自由水域) | 0 | 无惩罚，纯距离驱动 |

**设计原则：只惩罚错，不奖励对。** 正确航道代价恒为零，避免将路径过度拉向航道中心。

### 3.2 A* 步进代价

```
c(p_i, p_j; d) = ‖p_j - p_i‖  +  λ · c_lane(p_j; d)
                 └──距离──┘     └──航道惩罚──┘
```

A* 总评价函数：

```
f(n) = g(n) + w_h · h(n)

g(n) = Σ 步进代价  (从起点到当前节点)
h(n) = ‖p_n - p_goal‖  (欧氏距离启发项)

引导A*: w_h = 0.1  (弱化启发，增强探索)
传统A*: w_h = 1.0  (标准最优路径)
```

### 3.3 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `lane_direction` | +1 | 期望航向 (+1下游/-1上游) |
| `lane_lambda` | 5.0 | 航道合规权重，越大越强制走正确航道 |
| `map_resolution` | 0.5 | 栅格分辨率 (m/cell) |
| `use_lane_guidance` | true | 启用/禁用航道引导 |

### 3.4 动态调参

```bash
# 切换航向
rosparam set /ais_planner_node/lane_direction -1   # 逆流/上游/从右往左
rosparam set /ais_planner_node/lane_direction 1    # 顺流/下游/从左往右

# 调整航道约束强度
rosparam set /ais_planner_node/lane_lambda 10.0    # 强约束
rosparam set /ais_planner_node/lane_lambda 1.0     # 弱约束
```

## 四、航道约束路径优化

### 4.1 三目标优化

对A*原始路径顶点序列 P = {p₁, …, p_N}，最小化：

```
E(P) = ω_d · Σ‖p_i - p_i_raw‖²           (数据保真：不偏离A*参考)
     + ω_s · Σ‖p_{i-1} - 2p_i + p_{i+1}‖²  (几何平滑：抑制曲率震荡)
     + ω_v · Σ c_lane(p_i; d)              (航道偏离：惩罚走错航道)
```

### 4.2 梯度下降求解

数据保真项与平滑项为二次型，封闭求导。航道偏离项仅在路径点处于错误航道时产生梯度力：

```
∇c_lane(p_i) = -d · ∇Φ̄(p_i)   当 -d·Φ̄(p_i) > 0 (错误航道)
             = 0               当 -d·Φ̄(p_i) ≤ 0 (正确航道)
```

### 4.3 参数

| 参数 | 引导路径 | 传统路径 | 说明 |
|------|---------|---------|------|
| `weight_data` | 0.2 | 0.1 | 数据保真权重 |
| `weight_smooth` | 0.5 | 0.4 | 平滑权重 |
| `weight_lane` | 0.1 | 0.0 | 航道偏离权重 |
| `tolerance` | 0.0001 | 0.0001 | 收敛容差 |

## 六、已知限制与后续改进

1. **两级架构（Phase 2）**: 当前只有中层栅格A*，顶层河道网络图搜索尚未实现。多航段方向切换需手动调整 `lane_direction` 参数。

2. **惩罚-only代价**: 正确航道无正梯度引导，弱信号区域引导力不足。可考虑增加密度调制项（见plan文件步骤3）。

3. **局部方向场未用于权重**: 当前 `θ_local(p)` 仅用于诊断，KDE权重仍用全局参考方向。若未来需要更细粒度的空间方向感知，可将 `θ_local(p)` 作为逐点参考方向。
