# 基于异构图的初始分配(最小费用流) + 动态重分配(拍卖法) — 技术方案

## 1. 异构图建模

### 1.1 图定义

```
G = (V, E, τ, φ)

V = V_ship ∪ V_task ∪ V_road    (三种节点)
E = E_rr ∪ E_sr ∪ E_tr ∪ E_st   (四种边)
τ: V → {Ship, Task, Road}       (节点类型映射)
φ: 每种边类型的特征维度
```

### 1.2 节点定义

#### Ship 节点

```
特征向量 h_s ∈ R^8:
  [0] x                当前X坐标 (米)
  [1] y                当前Y坐标 (米)  
  [2] energy_ratio     能源比例 (0~1)
  [3] load_ratio       载重比例 (0~1)
  [4] health           健康度 (0~1)
  [5] speed_ratio      速度比例 (0~1)
  [6] is_idle          是否空闲 (0/1)
  [7] task_count       当前任务数 / 最大任务数
```

#### Task 节点

```
特征向量 h_t ∈ R^8:
  [0] pickup_x         装货港 X
  [1] pickup_y         装货港 Y
  [2] delivery_x       卸货港 X
  [3] delivery_y       卸货港 Y
  [4] payload_ratio    货物重量 / 最大载重
  [5] priority         优先级 (1/2/3)
  [6] urgency          时间紧急度
  [7] is_assigned      是否已分配 (0/1)
```

#### Road 节点 (路网节点)

```
特征向量 h_r ∈ R^5:
  [0] x                世界X坐标
  [1] y                世界Y坐标
  [2] is_port          是否港口 (0/1)
  [3] is_gas           是否加油站 (0/1)
  [4] degree           度数 (连接的边数)
```

### 1.3 边定义

| 边类型 | 连接 | 特征 | 作用 |
|--------|------|------|------|
| `road — road` | 路网邻接边 | `distance` (米), `traversable` (0/1) | 路网拓扑 |
| `ship — road` | 船 → 所在节点 | `is_current` (0/1) | 船定位 |
| `task — road` | 任务 → 装/卸货港 | `is_pickup` (0/1), `is_delivery` (0/1), `距离` | 任务定位 |
| `ship — task` | 船 → 候选任务 | **不存, 动态计算** | 分配计算 |

### 1.4 核心: Ship↔Task 边的动态构建

Ship↔Task 边不预先存储, 每次分配时动态计算:

```
edge_weight(ship_i, task_j) = cost(ship_i, task_j)

cost = d_pickup + d_exec + penalties
     = dist_matrix[ship.current_node, task.pickup_node]       # 当前位置→装货港
     + dist_matrix[task.pickup_node, task.delivery_node]      # 装货港→卸货港
     + overload_penalty(task.payload, ship.remaining_capacity) # 超载惩罚
     + energy_penalty(total_dist, ship.energy)                 # 能源不足惩罚
     + deadline_risk(task.deadline, current_time, total_dist, ship.speed)  # 延期风险
```

## 2. 初始分配: 最小费用流 (Min-Cost Flow)

### 2.1 为什么用最小费用流

| 对比 | 贪心 | 最小费用流 |
|------|------|-----------|
| 最优性 | 局部最优 | **全局最优** |
| 多船协调 | 顺序分配, 先到先得 | **同时考虑所有船** |
| 复杂度 | O(M×K) | O((M+K)³) ← 可接受 |
| 约束处理 | 硬编码检查 | **流网络天然处理** |

### 2.2 流网络构造

```
Source ──→ Ship_0 ──→ Task_0 ──→ Sink
   │                    ↑
   ├────→ Ship_1 ───────┤
   │                    │
   ├────→ Ship_2 ───────┤
   │                    │
   └────→ Ship_3 ───────┘

边定义:
  Source → Ship_i:    容量 = max_tasks_ship_can_handle
                      (ship.energy / min_task_energy)
                      费用 = 0

  Ship_i → Task_j:    容量 = 1 (每个任务只分配给一艘船)
                      费用 = cost(ship_i, task_j)
                      约束: 如果 task.payload > ship.remaining_capacity → 边不存在

  Task_j → Sink:      容量 = 1
                      费用 = 0

流程: 最小费用最大流 → 总流量 K = 分配的任务数
```

### 2.3 代价函数 (Ship→Task 边权重)

```python
def edge_cost(ship, task, dist_matrix, current_time):
    """
    计算 Ship_i → Task_j 边的费用
    对标异构图中的 Ship-Task 边权重
    """
    d_pickup  = dist_matrix[ship.current_node, task.pickup_node]    # A: 当前位置→装货港
    d_exec    = dist_matrix[task.pickup_node, task.delivery_node]   # B: 装货港→卸货港
    total_d   = d_pickup + d_exec
    
    # 1. 基础代价: 航行距离 (主要项)
    base = total_d / 1000.0  # 转为 km
    
    # 2. 载重约束惩罚
    if task.payload > ship.remaining_capacity:
        return float('inf')   # 不可行 → 边不存在
    load_penalty = max(0, (ship.load + task.payload) / ship.max_payload - 0.8) * 50
    
    # 3. 能源约束
    energy_need = total_d / 1000.0 * ship.energy_per_km
    if energy_need > ship.energy * 0.7:    # 超过70%余量 → 不可行
        return float('inf')
    energy_risk = max(0, energy_need - ship.energy * 0.3) * 0.1
    
    # 4. 时间紧急度
    if task.deadline < float('inf'):
        travel_time = total_d / ship.max_speed  # 秒
        time_to_complete = current_time + travel_time + 600  # +10min装卸
        if time_to_complete > task.deadline:      # 不可能在截止前完成
            return float('inf')
        urgency = max(0, 1.0 - (task.deadline - time_to_complete) / 3600.0)
    else:
        urgency = 0
    
    # 5. 负载均衡 (所有船的任务数尽量均匀)
    balance = len(ship.task_sequence) * 10
    
    return base + load_penalty + energy_risk + urgency * task.priority * 20 + balance
```

### 2.4 算法实现

```python
from collections import defaultdict
import heapq

def min_cost_flow_allocation(ships, tasks, dist_matrix, current_time=0):
    """
    最小费用流分配。
    
    返回: assignments = [(ship_id, task_id, cost), ...]
    """
    M = len(ships)
    K = len(tasks)
    
    # 节点编号:
    #   0 = Source
    #   1..M = Ship nodes
    #   M+1..M+K = Task nodes
    #   M+K+1 = Sink
    
    SOURCE = 0
    SINK = M + K + 1
    
    # 构建邻接表和容量/费用
    graph = defaultdict(list)  # u -> [(v, capacity, cost, rev_index)]
    
    def add_edge(u, v, cap, cost):
        graph[u].append([v, cap, cost, len(graph[v])])
        graph[v].append([u, 0, -cost, len(graph[u]) - 1])
    
    # Source → Ships (每个船最多执行的任务数)
    for i, ship in enumerate(ships):
        max_tasks = min(10, int(ship.energy / (0.1 * ship.max_energy)))  # 能源够做几个任务
        add_edge(SOURCE, 1 + i, max_tasks, 0)
    
    # Ships → Tasks
    for i, ship in enumerate(ships):
        for j, task in enumerate(tasks):
            cost = edge_cost(ship, task, dist_matrix, current_time)
            if cost < float('inf'):   # 可行才加边
                add_edge(1 + i, 1 + M + j, 1, cost)
    
    # Tasks → Sink
    for j in range(K):
        add_edge(1 + M + j, SINK, 1, 0)
    
    # ======= 最小费用流 (Successive Shortest Path) =======
    n_nodes = M + K + 2
    flow = 0
    cost = 0
    assignments = []
    
    while flow < K:  # 尝试分配所有任务
        # SPFA 找最短增广路
        dist = [float('inf')] * n_nodes
        dist[SOURCE] = 0
        prev_v = [-1] * n_nodes
        prev_e = [-1] * n_nodes
        in_queue = [False] * n_nodes
        
        q = deque([SOURCE])
        in_queue[SOURCE] = True
        
        while q:
            u = q.popleft()
            in_queue[u] = False
            for ei, (v, cap, w, _) in enumerate(graph[u]):
                if cap > 0 and dist[v] > dist[u] + w:
                    dist[v] = dist[u] + w
                    prev_v[v] = u
                    prev_e[v] = ei
                    if not in_queue[v]:
                        q.append(v)
                        in_queue[v] = True
        
        if dist[SINK] == float('inf'):
            break  # 无法分配更多
        
        # 沿最短路径推流
        d = 1  # 每条边容量都是1
        v = SINK
        while v != SOURCE:
            u = prev_v[v]
            ei = prev_e[v]
            graph[u][ei][1] -= d
            rev = graph[u][ei][3]
            graph[v][rev][1] += d
            v = u
        
        flow += d
        cost += dist[SINK] * d
        
        # 记录分配: 找到从 Source 到 Sink 经过的 Ship 和 Task
        for i in range(M):
            for ei, (v, cap, w, _) in enumerate(graph[1 + i]):
                if v > 1 + M and v <= 1 + M + K and cap == 0:
                    # 这条边有流量 → Ship i 分配了 Task (v - M - 1)
                    task_id = v - M - 1
                    assignments.append((i, task_id, w))
    
    # ======= 输出结果 =======
    # 每艘船按 pickup→delivery 距离排序任务 (2-opt 初始化)
    for ship_id in range(M):
        ship_tasks = [(tid, cost) for (sid, tid, cost) in assignments if sid == ship_id]
        # 按装货港距离排序
        ship_tasks.sort(key=lambda x: dist_matrix[ships[ship_id].current_node, 
                                                    tasks[x[0]].pickup_node])
        for rank, (tid, cost) in enumerate(ship_tasks):
            tasks[tid].assigned_ship = ship_id
            ships[ship_id].task_sequence.append(tid)
    
    return assignments, cost
```

### 2.5 复杂度分析

```
M = 船数, K = 任务数, N = 路网节点数

流网络节点数: M + K + 2
流网络边数: M×K (最坏情况)
SPFA 每次: O(E) = O(M×K)
总迭代次数: ≤K (每次增广1个任务)
总复杂度: O(K × M × K) = O(M×K²)

M=10, K=100 → 100,000 次操作 → < 0.1秒
M=50, K=500 → 12,500,000 → ~1秒 (可接受)
```

## 3. 动态重分配: 拍卖法 (Auction Algorithm)

### 3.1 为什么用拍卖法

| 对比 | 全量重跑最小费用流 | 拍卖法 |
|------|------------------|--------|
| 适用范围 | 静态/全局 | **增量/动态** |
| 计算量 | 每次全图重新求解 | **只改受影响的任务** |
| 收敛速度 | O(M×K²) | **O(K×log K)** 近似 |
| 实时性 | 慢 | **快, 适合实时** |
| 最优性 | 全局最优 | ε-近似最优 |

### 3.2 拍卖法原理

```
类比: 拍卖
  - 任务 = 物品 (每个物品需要被分配)
  - 船 = 竞拍者 (每个竞拍者想获得物品)
  - 价格 = 任务的当前拍卖价格 (初始为0)
  - 收益 = -(分配代价) - 价格

流程:
  1. 每个任务初始化价格 p_j = 0
  2. 每轮:
     a. 每艘船计算每个任务的收益: profit(s_i, t_j) = -cost(s_i, t_j) - p_j
     b. 选收益最高的任务 + 次高收益
     c. 出价: bid = profit_best - profit_second + ε
     d. 更新价格: p_j = p_j + bid
     e. 重新分配: 出价最高的船获得任务
  3. 价格收敛 → 分配稳定
```

### 3.3 重分配触发条件

```
条件1: 船的新增/退出
  船故障(释放任务) → 释放的任务进入拍卖池
  新船加入 → 参与竞拍

条件2: 任务的紧急度剧变
  dynamic_priority > base × 2 → 任务重新进入拍卖池
  截止时间迫近 → 提高任务"价值"

条件3: 船状态变化
  船能源突然下降 → 降低该船对所有任务的收益
  船提前完成 → 参与新任务竞拍

条件4: 全局代价漂移
  total_fleet_cost > initial_cost × 1.3 → 全局重拍
```

### 3.4 算法实现

```python
def auction_algorithm(ships, tasks, dist_matrix, 
                       epsilon=0.01, max_iterations=100,
                       fixed_assignments=None):
    """
    拍卖法进行动态任务分配。
    
    Args:
        ships: 所有船
        tasks: 所有任务 (含已分配和未分配)
        dist_matrix: 路网距离矩阵
        epsilon: 近似精度 (越小越精确, 越慢)
        max_iterations: 最大迭代次数
        fixed_assignments: 不需要重拍的任务集合 {(ship_id, task_id), ...}
    
    Returns:
        new_assignments: 新的分配结果
    """
    M = len(ships)
    K = len(tasks)
    
    # 初始化价格 (已分配任务的初始价格设得更高, 避免不必要的重拍)
    prices = np.zeros(K)
    if fixed_assignments:
        for sid, tid in fixed_assignments:
            # 这些任务价格设高, 其他船抢不走
            prices[tid] = 1000.0  
    
    # 当前分配状态
    assigned_to = {}  # task_id -> (ship_id, bid_price)
    
    def profit(ship_idx, task_idx):
        """船对任务的收益 = 负代价 - 价格"""
        c = edge_cost(ships[ship_idx], tasks[task_idx], dist_matrix, 0)
        if c == float('inf'):
            return -float('inf')
        return -c - prices[task_idx]
    
    # 构建收益矩阵 (缓存)
    profits = np.zeros((M, K))
    for i in range(M):
        for j in range(K):
            profits[i, j] = profit(i, j)
    
    for iteration in range(max_iterations):
        changed = False
        
        for i, ship in enumerate(ships):
            if ship.current_phase in ("loading", "unloading", "navigate_to_delivery"):
                # 正在执行关键任务的船不参与竞拍 (防止中断)
                continue
            
            # 找出收益最高和次高的任务
            best_task = -1
            best_profit = -float('inf')
            second_best = -float('inf')
            
            for j, task in enumerate(tasks):
                if task.status in ("completed", "cancelled"):
                    continue
                if (i, j) in (fixed_assignments or set()):
                    continue
                
                p = profits[i, j]
                if p > best_profit:
                    second_best = best_profit
                    best_profit = p
                    best_task = j
                elif p > second_best:
                    second_best = p
            
            if best_task < 0:
                continue
            
            # 计算出价
            if second_best == -float('inf'):
                bid = epsilon  # 没有竞争者
            else:
                bid = (best_profit - second_best) + epsilon
            
            # 更新价格
            old_price = prices[best_task]
            prices[best_task] += bid
            
            # 更新分配
            prev_owner = assigned_to.get(best_task, (-1, 0))[0]
            if prev_owner >= 0 and prev_owner != i:
                # 从旧船移除
                if best_task in ships[prev_owner].task_sequence:
                    ships[prev_owner].task_sequence.remove(best_task)
                ships[prev_owner].load -= tasks[best_task].payload
                tasks[best_task].assigned_ship = -1
            
            assigned_to[best_task] = (i, bid)
            
            # 给新船
            if best_task not in ships[i].task_sequence:
                ships[i].task_sequence.append(best_task)
            ships[i].load += tasks[best_task].payload
            tasks[best_task].assigned_ship = i
            tasks[best_task].reassign_count += 1
            
            # 更新收益矩阵 (价格变了)
            for ii in range(M):
                profits[ii, best_task] = profit(ii, best_task)
            
            changed = True
        
        if not changed:
            break  # 收敛
    
    # 构建分配结果
    new_assignments = []
    for task_id, (ship_id, bid) in assigned_to.items():
        new_assignments.append((ship_id, task_id, bid))
    
    return new_assignments, prices
```

### 3.5 重分配场景示例

```
场景: Ship_0 能源紧急 (energy_ratio = 8%)

拍卖过程:
  Ship_0 的 profit 函数对所有任务大幅下降 (因为 energy_penalty 极高)
  → Ship_0 不再能"竞得"自己的任务
  → Ship_1, Ship_2, Ship_3 出价更高, 抢走这些任务
  → Ship_0 的任务被自动转移到其他船

不需要显式"释放"逻辑 — 拍卖自然处理

例:
  T5 原本在 Ship_0, price=0, profit(Ship_0,T5)=-8, profit(Ship_1,T5)=-5
  Ship_0 能源下降 → profit(Ship_0,T5)=-500 (不可行)
  → Ship_1 竞得 T5 (bid=ε), price[T5]=ε
  → T5 自动转移到 Ship_1
```

## 4. 任务排序 (分配后的序列优化)

### 4.1 问题

最小费用流给每艘船分配了一组任务, 但这些任务的执行顺序还没优化。

### 4.2 方法: 2-opt + 插入排序

```python
def optimize_execution_order(ship, tasks, dist_matrix):
    """
    对于已分配的任务集合, 优化执行顺序。
    最小化总航程: ship_node → T0.pickup → T0.delivery → T1.pickup → ...
    """
    task_list = [t for t in tasks if t.id in ship.task_sequence]
    
    # 初始顺序: 按装货港距离
    task_list.sort(key=lambda t: dist_matrix[ship.current_node, t.pickup_node])
    
    # 2-opt 局部优化
    improved = True
    while improved:
        improved = False
        for i in range(len(task_list)):
            for j in range(i + 2, len(task_list)):
                if swap_reduces_cost(task_list, i, j, ship, dist_matrix):
                    task_list[i:j+1] = reversed(task_list[i:j+1])
                    improved = True
    
    ship.execution_order = [t.id for t in task_list]
```

## 5. 完整流程

```
┌─────────────────────────────────────────────────────────┐
│                Phase 1: 初始分配                         │
│                                                          │
│  异构图构建 → Ship-Task 边代价计算                        │
│  → 最小费用流求解 → 全局最优分配                          │
│  → 2-opt 序列优化                                        │
│                                                          │
│  输出: 每艘船的任务序列 + 总代价                          │
└──────────────────────┬──────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────┐
│                Phase 2: 执行监控                         │
│                                                          │
│  每个调度周期:                                            │
│    1. 更新船状态 (位置/能源/进度)                         │
│    2. 更新任务动态优先级                                  │
│    3. 检测重分配触发条件                                  │
│    4. 如果触发 → 固定关键任务, 其余进入拍卖池             │
└──────────────────────┬──────────────────────────────────┘
                       ↓ (触发重分配时)
┌─────────────────────────────────────────────────────────┐
│                Phase 3: 拍卖重分配                       │
│                                                          │
│  1. 标记不可动任务:                                        │
│     - 正在装货/卸货 (loading/unloading)                  │
│     - 在途已装货 (navigate_to_delivery)                  │
│                                                          │
│  2. 拍卖池: 其余所有未开始任务  + 新到达任务              │
│                                                          │
│  3. ε-拍卖法求解 → 更新分配                              │
│                                                          │
│  4. 检查总代价变化:                                       │
│     if delta_cost > threshold: 接受新分配                 │
│     else: 保持原分配                                      │
│                                                          │
│  输出: 重分配前后对比 + 代价变化                           │
└─────────────────────────────────────────────────────────┘
```
