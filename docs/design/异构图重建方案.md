# 异构图重建方案（v2）

## 设计原则

> **节点只表示"实体自身状态"，边表示"实体之间的关系"。**

- 当前位置、到任务距离、预计完成时间 → **边属性**，不放节点
- 路网节点不再区分 Port/Gas/Waypoint → 统一 RoadNode + Type 字段
- Ship↔Task 不是简单的分配关系，而是 Candidate / Assigned / Executing / Finished 四种

---

## 1. 节点设计

### 1.1 Ship 节点（每艘船一个）

```
Ship 节点特征 (8维):
┌─────────────────────────────────────────────────────┐
│ [0] max_payload      最大载重 (吨)       静态        │
│ [1] max_speed        最大航速 (m/s)      静态        │
│ [2] max_energy       最大能源 (kWh)      静态        │
│ [3] remaining_energy 剩余能源 (kWh)      动态 ✓      │
│ [4] current_load     当前载货 (吨)       动态 ✓      │
│ [5] health           健康度 (0~1)        动态 ✓      │
│ [6] status           状态编码            动态 ✓      │
│                       0=Idle 1=Moving               │
│                       2=Loading 3=Transporting       │
│ [7] task_count       当前任务数           动态 ✓      │
└─────────────────────────────────────────────────────┘

不放:
  ✗ 当前位置 → 在 Ship→Road 边上
  ✗ ETA → 在 Ship→Road 边上
  ✗ 到任务距离 → 在 Ship→Task 边上
  ✗ 能耗 → 在 Road→Road 边上
```

### 1.2 RoadNode 节点（统一，不再区分 Port/Gas/Waypoint）

```
RoadNode 节点特征 (8维):
┌─────────────────────────────────────────────────────┐
│ [0] x                世界X坐标 (归一化)   静态       │
│ [1] y                世界Y坐标 (归一化)   静态       │
│ [2] node_type        类型编码             静态       │
│                       0=Normal 1=Port                │
│                       2=GasStation                   │
│ [3] degree           连接度               静态       │
│ [4] capacity         容量 (港口泊位数)    半静态     │
│ [5] congestion       当前拥堵 (0~1)       动态 ✓     │
│ [6] waiting_time     等待时间 (秒)        动态 ✓     │
│ [7] reserved         保留                     │
└─────────────────────────────────────────────────────┘
```

### 1.3 Task 节点

```
Task 节点特征 (8维):
┌─────────────────────────────────────────────────────┐
│ [0] priority         优先级 (1/2/3)        静态       │
│ [1] payload          货物重量 (吨)         静态       │
│ [2] deadline         截止时间 (归一化)     静态       │
│ [3] remaining_time   剩余时间 (归一化)     动态 ✓     │
│ [4] reward           完成收益             静态       │
│ [5] status           状态编码             动态 ✓     │
│                       0=Pending 1=Assigned           │
│                       2=Executing 3=Finished         │
│ [6] dynamic_priority 动态优先级           动态 ✓     │
│ [7] reassign_count   重分配次数           动态 ✓     │
└─────────────────────────────────────────────────────┘

不放:
  ✗ pickup_node → 在 Task→Road 边上
  ✗ delivery_node → 在 Task→Road 边上
  ✗ assigned_ship → 在 Ship→Task 边上
```

## 2. 边设计（5种）

### 2.1 Road↔Road 边（航道）

```
Road_i ──[connects_to]──→ Road_j

边特征 (6维):
┌─────────────────────────────────────────────────────┐
│ [0] length           航道长度 (米)         静态       │
│ [1] travel_time      航行时间 (秒)        动态 ✓     │
│                       = length / max_speed           │
│ [2] energy_cost      能耗 (kWh)           动态 ✓     │
│                       = length × energy_rate         │
│ [3] width            航道宽度 (米)        静态       │
│ [4] congestion       拥堵程度 (0~1)       动态 ✓     │
│ [5] risk             风险 (0~1)           半静态     │
└─────────────────────────────────────────────────────┘

● 无向边 × 149条 (来自现有路网)
● 权重: 像素距离 / scale_factor
```

### 2.2 Ship→Road 边（船定位）

```
Ship_i ──[located_at]──→ Road_j

边特征 (3维):
┌─────────────────────────────────────────────────────┐
│ [0] distance         到节点的距离 (米)     动态 ✓     │
│                       0 = 就在此节点                 │
│ [1] arrival_time     预计到达时间 (秒)    动态 ✓     │
│ [2] is_current       是否当前所在 (0/1)   动态 ✓     │
└─────────────────────────────────────────────────────┘

● 每艘船 × 1条 = M条
● 船不在路网精确节点上时, distance > 0
```

### 2.3 Task→Road 边（任务定位）

```
Task_k ──[pickup_at]──→ Road_p
Task_k ──[delivery_at]──→ Road_d

边特征 (3维):
┌─────────────────────────────────────────────────────┐
│ [0] edge_type        0=pickup 1=delivery   静态       │
│ [1] load_weight      装卸重量 (吨)         静态       │
│                       装货=+payload 卸货=-payload     │
│ [2] operation_time   装卸时间 (秒)         静态       │
│                       装货≈300s 卸货≈180s            │
└─────────────────────────────────────────────────────┘

● 每个任务 × 2条 = 2K条
● 基本静态, 只在任务创建时建立
```

### 2.4 Ship↔Task 边（核心 — 分配关系）

```
四种关系类型:

  Ship_i ──[candidate]──→ Task_k    (可以分配给此船)
  Ship_i ──[assigned]───→ Task_k    (已分配)
  Ship_i ──[executing]──→ Task_k    (正在执行)
  Ship_i ──[finished]───→ Task_k    (已完成)

边特征 (6维):
┌─────────────────────────────────────────────────────┐
│ [0] relation_type    0=candidate 1=assigned          │
│                       2=executing 3=finished          │
│ [1] estimated_cost   预估总代价            动态 ✓     │
│                       = travel_dist + exec_dist      │
│                         + time_penalty + energy_risk  │
│ [2] travel_distance  到装货港距离 (米)     动态 ✓     │
│ [3] execution_dist   装货港→卸货港 (米)    静态       │
│ [4] estimated_time   预估完成时间 (秒)    动态 ✓     │
│ [5] energy_required  预估能耗 (kWh)       动态 ✓     │
│ [6] match_score      匹配得分 (0~1)       动态 ✓     │
│                       = f(cost, deadline, capacity)   │
└─────────────────────────────────────────────────────┘

● 最多 M×K 条 candidate 边 (每艘船×每个未分配任务)
● assigned 边 = 已分配的任务数
● executing 边 ≤ 每艘船1条 (当前任务)
● candidate 边每周期动态重建 (状态变了代价也变)
```

### 2.5 Task↔Task 边（新增 ⭐）

```
Task_k ──[nearby]──→ Task_m        (空间邻近)
Task_k ──[same_pickup]──→ Task_m   (同装货港)
Task_k ──[same_delivery]──→ Task_m (同卸货港)
Task_k ──[depends_on]──→ Task_m    (必须k先完成)

边特征 (4维):
┌─────────────────────────────────────────────────────┐
│ [0] relation_type    0=nearby 1=same_pickup          │
│                       2=same_delivery 3=depends       │
│ [1] spatial_dist     两任务装货港间距离 (米)          │
│ [2] time_gap         截止时间差 (秒)                  │
│ [3] priority_gap     优先级差                        │
└─────────────────────────────────────────────────────┘

nearby 阈值: 两任务装货港距离 < 500m → 建立边
same_pickup/delivery: 直接比较 node_id
depends_on: 同艘船连续任务 T_k → T_{k+1}

● nearby: O(K²) 但可裁剪 (每个任务最多5条 nearby 边)
● same_pickup/delivery: 自动从 Task→Road 边推导
● depends_on: 从 Ship→Task assigned 边推导
```

## 3. 完整图结构

```
                    ┌──────────────┐
                    │   Ship (M)   │
                    │  8维特征      │
                    └──┬───┬───┬──┘
          located_at  │   │   │  candidate/assigned/executing
                      │   │   └──────────────────────┐
                      │   │                           │
         ┌────────────┘   │    ┌──────────────────┐   │
         ▼                │    │   Task (K)       │   │
   ┌──────────┐           │    │   8维特征         │   │
   │  Road    │◄──────────┘    └──┬───────┬───────┘   │
   │  (N)     │   pickup/        │       │            │
   │  8维特征  │   delivery       │  nearby/same/      │
   └────┬─────┘                  │  depends          │
        │                        ▼                   │
        │ connects_to      ┌──────────┐              │
        └─────────────────►│  Task    │◄─────────────┘
                           │  (K)     │
                           └──────────┘

边统计:
  Road↔Road:   149 条  (静态路网)
  Ship→Road:   4 条    (M条, 动态)
  Task→Road:   2K 条   (每任务2条, 基本静态)
  Ship→Task:   ≤M×K 条 (candidate: 动态重建, assigned/executing: 少量)
  Task→Task:   ≤5K 条  (nearby: 裁剪到5条/任务)
```

## 4. 数据结构改动

### 4.1 hetero_graph.py 改动

```python
class HeteroGraphV2:
    """重构后的异构图"""
    M: int  # 船数
    K: int  # 任务数
    N: int  # 路网节点数

    # ── 节点特征 ──
    ship_x: np.ndarray    # (M, 8)
    road_x: np.ndarray    # (N, 8)
    task_x: np.ndarray    # (K, 8)

    # ── 边索引 (CSR格式) ──
    # Road↔Road
    rr_edges: np.ndarray       # (2, E_rr)
    rr_feat: np.ndarray        # (E_rr, 6)

    # Ship→Road
    sr_edges: np.ndarray       # (2, M)
    sr_feat: np.ndarray        # (M, 3)

    # Task→Road (pickup + delivery)
    tr_edges: np.ndarray       # (2, 2K)
    tr_feat: np.ndarray        # (2K, 3)

    # Ship→Task (candidate)
    st_edges: np.ndarray       # (2, E_st)
    st_feat: np.ndarray        # (E_st, 7)

    # Task→Task (nearby/same/depends)
    tt_edges: np.ndarray       # (2, E_tt)
    tt_feat: np.ndarray        # (E_tt, 4)
```

### 4.2 代价计算改动

```
旧: cost(ship, task) = 函数内计算所有项
新: cost = Ship→Task 边上的 estimated_cost
     = travel_distance + execution_dist + penalties

边属性直接提供:
  - min_cost_flow:     读 st_feat[j].estimated_cost 作为边权重
  - auction_realloc:   读 st_feat[j].match_score 作为收益基准
  - HGT attention:    用边类型 + 边特征做消息传递
```

### 4.3 动态更新

```
每个调度周期:
  1. 更新 Ship 节点: energy, load, health, status, task_count
  2. 更新 Road 节点: congestion, waiting_time
  3. 更新 Task 节点: remaining_time, dynamic_priority, status
  4. 更新 Ship→Road 边: distance, arrival_time
  5. 重建 Ship→Task candidate 边: 状态变 → 代价变
  6. 更新 Road→Road 边: travel_time, congestion
  7. 更新 Task→Task 边: time_gap (截止时间变了)
```

## 5. 改动影响

| 模块 | 改动 |
|------|------|
| `hetero_graph.py` | 重写: 5种边 + 边特征 + 动态更新 |
| `min_cost_flow.py` | Ship→Task cost 不再每次计算, 直接读 `st_feat.estimated_cost` |
| `auction_realloc.py` | profit 改用 `st_feat.match_score` + `st_feat.estimated_cost` |
| `visualize_*.py` | 增加 Task→Task 边可视化 |
| `real_time_scheduler.py` | 每周期调用 `hetero_graph.update()` 而非重建 |

## 6. 对比

| | 旧版 (v1) | 新版 (v2) |
|------|---------|---------|
| 节点类型 | Ship/Task/Road (分 Port/Gas) | Ship/Task/RoadNode (统一 Type) |
| RoadNode | Port≠Gas≠Normal | **统一 + Type字段** |
| Task节点 | 含 pickup_node/delivery_node | **不含, 在 Task→Road 边上** |
| Ship节点 | 含位置信息 | **不含, 在 Ship→Road 边上** |
| 边类型 | 4种 (无 Task→Task) | **5种 (含 Task→Task)** |
| Ship→Task边 | 动态计算, 不存储 | **存储为 candidate/assigned/executing/finished** |
| Road→Road边 | 仅距离 | **距离+时间+能耗+拥堵+风险** |
| 边特征 | 无 | **每条边有3~7维特征** |
| 代价计算 | 函数内计算 | **直接读边上的 estimated_cost** |
