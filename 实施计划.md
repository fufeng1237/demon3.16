## 二、模块职责

### 模块1: hetero_graph.py（异构图）

```
职责: 维护 Ship-Road-Task 异构图

输入: ships, tasks, road_network
输出: HeteroGraph 对象

核心方法:
  build_hetero_graph()   — 初始构建整个图
  update_graph()         — 每周期更新动态属性
  get_ship_task_edge()   — 查询 Ship→Task 边
  get_road_neighbors()   — 查询路网邻接
```

### 模块2: graph_evaluator.py（图评价器）

```
职责: 计算 Ship-Task 匹配评分

输入: HeteroGraph, ships, tasks
输出: 每对 (ship, task) 的匹配分数

核心方法:
  evaluate(ship, task)        — 单对评分
  get_top_k(ship, tasks, K)   — Top-K 候选
  compute_insert_cost()       — 插入代价
  check_feasibility()          — 约束检查
```

### 模块3: alns_scheduler.py（ALNS 优化器）

```
职责: 生成和优化运输任务队列

输入: ships, tasks, graph_evaluator
输出: 每艘船的 Route (有序任务队列)

核心方法:
  initial_solution()     — 贪心初始解
  destroy()              — 破坏算子
  repair()               — 修复算子
  alns_loop()            — SA 主循环
  validate_route()       — 约束验证
```

### 模块4: route_executor.py（执行器）

```
职责: 按 Route 执行运输任务

输入: ships, routes, graph
输出: 更新后的 ships, tasks, graph

核心方法:
  execute_step(dt)       — 推进一个时间步
  advance_ship(ship)     — 推进单船
  start_task()           — 开始新任务
  complete_task()        — 完成任务
```

### 模块5: rolling_horizon.py（滚动优化）

```
职责: 周期性重优化未来任务

输入: ships, tasks, graph, frozen_count=N
输出: 优化后的 Routes

核心方法:
  optimize_future()      — 优化未冻结任务
  should_trigger()       — 判断是否触发
  freeze_front(N)        — 冻结前N个任务
```

### 模块6: event_detector.py（状态检测）

```
职责: 检测需要重调度的异常

输入: ships, tasks, graph
输出: Event 列表

核心方法:
  detect()               — 综合检测
  detect_ship()          — 船检测
  detect_task()          — 任务检测
```

### 模块7: route_repair.py（路由修复）

```
职责: 局部修复受影响的任务

输入: ships, tasks, events, graph
输出: 更新后的 Routes

核心方法:
  release_affected()     — 释放受影响任务
  repair()               — 局部修复
  merge()                — 合并结果
```

### 模块8: scheduler.py（主调度器）

```
职责: 整合所有模块, 提供统一入口

核心方法:
  initialize()           — 初始调度
  step()                 — 执行一个时间步
  run()                  — 持续运行
  handle_event()         — 事件处理
```

## 三、数据流

```
scheduler.initialize():
  road_network ──→ hetero_graph.build() ──→ HeteroGraph
  HeteroGraph ──→ graph_evaluator ──→ Ship-Task scores
  scores ──→ alns_scheduler ──→ Routes (每艘船的任务队列)

scheduler.step():
  Routes ──→ route_executor.execute_step() ──→ 更新 ships/tasks
  更新的数据 ──→ hetero_graph.update() ──→ 新 HeteroGraph
  event_detector.detect() ──→ Events

scheduler.handle_event():
  Events ──→ route_repair.release_affected() ──→ 释放的任务
  释放的任务 ──→ graph_evaluator (重算评分) ──→ alns_scheduler (局部优化)
  ──→ 更新 Routes

rolling_horizon.optimize_future():
  未冻结任务 ──→ graph_evaluator ──→ alns_scheduler (局部优化)
  ──→ 更新 Routes
```

| 步骤 | 文件 | 内容 |

## 五、关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| Evaluator 冷启动 | 规则版 (InsertCost + 约束) | 无需训练数据, 立即可用 |
| ALNS 初始解 | Cheapest Insertion | 简单快速, ALNS 后续会优化 |
| 冻结任务数 | N=2 | 正在执行的 + 下一个, 防止中断 |
| Rolling Horizon 触发 | 每完成一个任务 | 粒度适中 |
| Route Repair 触发 | 异常事件 | 避免频繁全局重规划 |
| HGT 集成时机 | Phase 4 | 需要先收集运行数据训练 |
