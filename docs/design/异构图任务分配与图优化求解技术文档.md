# 异构图任务分配与图优化求解技术文档

## 1. 目标

建立 **Ship--Road--Task**
异构图，将任务分配与动态重分配统一表示为图优化问题，而不是传统
Greedy+2-opt。

------------------------------------------------------------------------

## 2. 异构图建模

### 节点类型

  节点   主要属性
  ------ --------------------------------------------
  Ship   位置、速度、电量、载重、预计完成时间、状态
  Road   坐标、类型、拥堵、容量
  Task   起点、终点、截止时间、优先级、载荷

### 边类型

-   Ship → Road：LocatedAt
-   Ship → Task：Candidate / Assigned / Executing
-   Task → Road：Pickup / Delivery
-   Road ↔ Road：Connected
-   Task ↔ Task：Dependency / Near
-   Ship ↔ Ship：Communication / Relay

## 4. 各方法简介

### 4.1 Min-Cost Flow

建立 Ship→Task
二部图，以距离、时间、电量、风险组成边权，求解总代价最小流。 **优点：**
全局最优、速度快、工业界成熟。

### 4.2 Hungarian

求解一对一最优匹配，复杂度 O(n³)。

### 4.3 Auction

任务竞拍，适合新增任务、船舶故障后的在线重分配，仅局部更新。

### 4.4 MILP

建立0-1决策变量，将容量、时间窗、电量等约束统一优化，可获得严格最优，但规模大时计算较慢。

### 4.5 Maximum Matching

根据Ship-Task评分进行最大权匹配。

### 4.6 Steiner Tree

寻找覆盖多个任务点的最短运输骨架，减少重复航行。

### 4.7 VRP系列

包括LKH、ALNS、Tabu Search、OR-Tools等，用于优化单船任务执行顺序。

### 4.8 Conflict Graph

建立港口、航道等资源冲突图，实现局部冲突消解。

### 4.9 Space-Time Graph

节点扩展为"位置+时间"，适合截止时间、等待、排队建模。

### 4.10 Hypergraph

一条超边可同时连接船舶、任务、港口、天气等多个实体。

## 5. 推荐总体架构

``` text
Ship + Road + Task
        │
 Heterogeneous Graph
        │
   HGT/HAN Encoder
        │
 Ship-Task Score
        │
 ┌───────────────┐
 │ 初始分配      │ Min-Cost Flow
 │ 动态重分配    │ Auction + 局部Min-Cost Flow
 │ 船内排序      │ OR-Tools / ALNS / LKH
 └───────────────┘
        │
      执行
        │
   故障/新增任务
        │
     局部更新图
```
