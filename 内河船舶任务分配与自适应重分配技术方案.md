# 基于异构图神经网络的内河船舶任务分配与自适应重分配技术方案

## 1. 系统概述

### 1.1 目标

构建面向内河运输场景的动态任务分配系统，实现：

- **多船多任务初始分配**：给定 M 艘船和 K 个运输任务，求解每艘船负责哪些任务及执行顺序，最小化总航行代价
- **运行过程中的实时自适应重分配**：当发生船舶故障、能源不足、提前完成、任务延期等事件时，在线调整分配方案

系统暂不考虑底层路径规划（由 ROS 导航栈处理），仅关注顶层任务调度。

### 1.2 核心挑战

| 挑战 | 说明 |
|------|------|
| 一对多分配 | 每艘船可执行多个任务（M < K），非一对一匹配 |
| 路网拓扑约束 | 船必须沿河道行驶，不能走直线 |
| 动态适应性 | 船故障、新任务到达、能源不足时需在线重规划 |
| 异构信息融合 | 船状态、任务属性、路网结构是不同类型的信息源 |

### 1.3 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    ROS 任务调度节点                       │
├─────────────────────────────────────────────────────────┤
│  ┌──────────┐   ┌──────────┐   ┌──────────┐            │
│  │ 状态感知  │ → │ 异构图构建 │ → │ HGT 编码  │            │
│  │(ROS topics)│   │(Graph)    │   │(Encoder)  │            │
│  └──────────┘   └──────────┘   └──────────┘            │
│                                       ↓                  │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐            │
│  │ 任务执行  │ ← │ 重分配决策 │ ← │ PPO 分配  │            │
│  │(ROS action)│  │(Gate)     │   │(Policy)   │            │
│  └──────────┘   └──────────┘   └──────────┘            │
│       ↓                                                  │
│  ┌──────────────────────────────────────┐               │
│  │         事件监测 (Event Monitor)       │               │
```

## 2. 异构动态图建模

### 2.1 图定义

设动态图为：

$$G_t = (V_t, E_t)$$

包含三种节点类型和四种边类型。

### 2.2 Ship 节点（船舶）

|------|------|

### 2.5 边关系

| 边类型 | 连接 | 特征 | 用途 |
|--------|------|------|------|
| ROAD↔ROAD | 路网节点之间 | 距离、时间、能耗 | 路网拓扑传播 |
| SHIP→ROAD | 船到其所在路网节点 | 当前位置关联 | 船定位 |
| TASK→ROAD | 任务到其装/卸货节点 | 取货/送货关联 | 任务定位 |
| SHIP↔TASK | 船到候选任务 | 路网距离、预计时间、能耗、载重可行性 | 分配决策 |

## 3. HGT 异构图编码器

### 3.1 设计原理

采用 **Heterogeneous Graph Transformer (HGT)**，对每种边类型使用独立的多头注意力机制，并保留本方案的 **核心创新：路网距离偏置注意力**。

### 3.2 路网距离偏置注意力（核心创新）

标准 Attention：
$$\alpha = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)$$

本方案 Attention：
$$\alpha = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} - \lambda \cdot D_{roadmap}\right)$$

其中 $\lambda$ 是可学习参数，$D_{roadmap}$ 是预计算的 A* 路网最短路径距离矩阵。

**作用**：在船-任务注意力中，路网距离远的配对被天然压制，模型在训练初期就具有路网距离感知能力。

### 3.3 HGT 层结构

```
输入: h_r(N,d), h_s(M,d), h_t(K,d)

边类型1: ROAD ← ROAD (多头自注意力 + 邻接mask)
  Q,K,V = W_q(h_r), W_k(h_r), W_v(h_r)
  mask = 路网邻接矩阵（-inf断开, 0连通, 含自环）
  msg_road = MultiHeadAttention(Q,K,V, mask)

边类型2: SHIP ← ROAD (线性聚合，船查询所在路网节点)
  msg_ship_road = adj_sr · W_sr(h_r)

边类型3: TASK ← ROAD (线性聚合，任务查询装/卸货港口)
  msg_task_road = adj_tr · W_tr(h_r)

边类型4: SHIP ← TASK (路网距离偏置多头交叉注意力)
  msg_ship_task = MultiHeadRoadmapAttention(h_s, h_t, D_st)

聚合 (A-Linear + 残差 + LayerNorm):
  h_r' = LayerNorm(h_r + α_r · A_road(msg_road))
  h_s' = LayerNorm(h_s + α_s · A_ship(msg_ship_road + msg_ship_task))
  h_t' = LayerNorm(h_t + α_t · A_task(msg_task_road))
```

### 3.4 编码器参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| d_hidden | 64~128 | 隐藏维度 |
| n_layers | 2~3 | HGT 层数 |
| n_heads | 4~8 | 注意力头数 |
| λ_init | 0.5 | 路网偏置初始强度 |

## 4. PPO 任务分配策略

### 4.1 状态空间

将 HGT 编码后的图嵌入作为状态：

$$s = \left[\frac{1}{M}\sum_{i} h_s^{(i)},\ \frac{1}{K}\sum_{k} h_t^{(k)}\right] \in \mathbb{R}^{2d}$$

### 4.2 动作空间

离散动作：选择 (船i, 任务k) 配对，共 $M \times K$ 个候选。

通过容量掩码过滤无效动作：
- 任务已分配 → mask
- 任务重量超过船剩余载重 → mask

### 4.3 分配架构

```
h_s(M,d) ──→ Cross-Attention(路网偏置) ──→ Bilinear(h_s', h_t) ──→ Logits(M,K)
h_t(K,d) ──→                                                      → Softmax → 采样(ship, task)
D_st(M,K) ──→ 路网距离偏置                                        → 容量Mask
```

### 4.4 任务排序（Pointer Network）

分配后，每艘船的任务通过 Pointer Network 确定执行顺序：

```
船i的任务序列 → LSTM Encoder → Pointer Decoder(含路网TSP距离偏置) → 访问顺序
```

Pointer 得分引入路网距离偏置：
$$u_j = v^T \cdot \tanh(W_{ref} e_j + W_{qry} d_t) - \beta \cdot D_{roadmap}[prev, j]$$

### 4.5 奖励函数

多目标加权奖励：

$$R = 0.30 \cdot R_{dist} + 0.10 \cdot R_{energy} + 0.25 \cdot R_{complete} + 0.10 \cdot R_{constraint} + 0.05 \cdot R_{balance} + 0.10 \cdot R_{fault} + 0.05 \cdot R_{gate} + 0.05 \cdot R_{new}$$

| 分量 | 公式 | 说明 |
|------|------|------|
| R_dist | $-total\_dist / baseline\_dist$ | 距离效率 |
| R_energy | $-total\_energy / energy\_budget$ | 能源效率 |
| R_complete | $completed / total$ | 任务完成率 |
| R_constraint | $-10 \times violations$ | 约束违反惩罚 |
| R_balance | $-std(ship\_loads)$ | 负载均衡 |
| R_fault | $+bonus$ | 故障恢复奖励 |
| R_gate | $-0.1 \times n\_realloc$ | 不必要的重分配惩罚 |
| R_new | $+bonus$ | 新任务处理奖励 |

### 4.6 训练流程

```
Phase 1: 监督预训练 (Imitation Learning)
  ┌─ 标签生成: 贪心 + 2-opt 启发式求解器 ─┐
  │  损失: CrossEntropy(分配) + 0.3×CrossEntropy(排序)  │
  │  1500 epochs × 2 scenarios                           │
  └─ 输出: 99%+ 质量比的初始分配模型 ────────────────────┘
                          ↓
Phase 2: PPO 强化学习微调
  ┌─ Actor: 编码器 + 分配器 │ Critic: ValueNetwork ─┐
  │  500 episodes, GAE 优势估计, Clipped Surrogate    │
  │  冻结 GNN 编码器 (freeze_gnn=True)                │
  └─ 输出: 在线自适应的策略网络 ───────────────────────┘
                          ↓
Phase 3: 偏差门控训练
  ┌─ 正样本(有事件→应触发) 500个  ─┐
  │  负样本(正常状态→不触发) 500个  │
  │  BCE 损失, 500 epochs         │
  └─ 输出: 学习型重分配触发器 ──────┘
```

## 5. 自适应重分配

### 5.1 事件驱动架构

采用 **事件驱动 (Event-triggered Rescheduling)** 机制。

监测节点持续监听船队状态，当偏差超过学习阈值时触发重分配。

### 5.2 偏差向量（每船 9 维）

| 维度 | 名称 | 含义 | 计算方式 |
|------|------|------|----------|
| 0 | energy_dev | 能源偏差 | 实际能耗 / 预期能耗 - 1 |
| 1 | progress_dev | 进度偏差 | 1 - 已完成 / 预期完成 |
| 2 | health_dev | 健康偏差 | 1 - 当前健康度 |
| 3 | capacity_dev | 容量偏差 | 当前负载 / 最大载重 - 0.9 |
| 4 | has_unassigned | 存在未分配 | 0/1 全局标志 |
| 5 | has_congestion | 存在拥堵 | 0/1 全局标志 |
| 6 | has_collision | 存在碰撞 | 0/1 全局标志 |
| 7 | energy_shortage | 能源不足 | 0/1 全局标志（任船 < 20%） |
| 8 | task_delayed | 任务延期 | 0/1 全局标志（任任务超 deadline） |

### 5.3 偏差门控网络 (DeviationGate)

```
dev_matrix(M, 9)
    → mean_pool + max_pool → dev_vec(18)
    → ┌─ Gate分支: FC(18→32) → Tanh → FC(32→1) → Sigmoid → trigger_prob (0~1)
    → └─ Scope分支: FC(18→32) → Tanh → FC(32→3) → Softmax → [p_local, p_ship, p_global]

决策: trigger_prob > 0.5 则触发重分配
范围: argmax(p_local, p_ship, p_global)
```

### 5.4 四种重分配事件

#### 事件1：船舶故障

```
触发条件: 船i 健康度骤降至 < 0.3

决策逻辑:
  if 未装货:
    释放该船全部任务 → 重新分配给其他健康船
  if 已装货:
    方案A: 最近港口卸货 → 其他船接力
    方案B: 等待维修 → 继续执行
    Cost = Time_cost + Delay_cost + Transfer_cost
    选择 min(Cost_A, Cost_B)

重分配范围: LOCAL（从故障船取回任务）
```

#### 事件2：能源不足

```
触发条件: 船i 能源降至 max_energy × 0.2 以下

决策逻辑:
  方案A (加油): 前往最近加油站补给后继续
    Cost_A = dist(当前位置→加油站) × (1 + refuel_penalty)
  
  方案B (移交): 释放未开始任务，交由其他船执行
    Cost_B = Σ min_dist(其他船→任务取货点) × (1 + transfer_penalty)
  
  选择 min(Cost_A, Cost_B)

重分配范围: LOCAL（从缺能船取回任务）
```

#### 事件3：提前完成任务

```
触发条件: 船i 在预期时间前完成全部任务

响应:
  - 船状态 → Idle
  - 参与剩余未分配任务的竞争
  - 执行局部增量调度（不改变已分配的任务）

重分配范围: 通常不触发 Gate（偏差小），或者 SHIP 级别
```

#### 事件4：任务延期

```
触发条件: 任务的 ETA 超过其 deadline，或超过阈值

决策逻辑:
  方案A (继续): 当前船继续执行，但付出延期惩罚
    Cost_A = 剩余距离 + Priority × deadline_penalty
  
  方案B (重分配): 转移任务至最快可用船
    Cost_B = dist(最快船→取货点) + dist(取货→送货) × (1 + transfer_penalty)
  
  选择 min(Cost_A, Cost_B)

重分配范围: LOCAL 或 SHIP（取决于延期任务数）
```

```python
    def __init__(self):
        # 加载模型
        self.encoder = HGTEncoder(d_hidden=64, n_layers=2, n_heads=4)
        self.assigner = RLTaskAssigner(d_hidden=64)
        self.pointer = PointerNetwork(d_hidden=64)
        self.gate = DeviationGate(d_hidden=32, n_dev_features=9)
        self.load_model('model.pt')

        # ROS 接口
        self.state_sub = ...     # 订阅船舶状态
        self.task_sub = ...      # 订阅新任务
        self.cmd_pub = ...       # 发布分配指令

        # 定时器: 每个调度周期
        self.timer = self.create_timer(1.0, self.scheduling_loop)

        # 状态缓存
        self.ships = []
        self.tasks = []
        self.assignments = {}

    def scheduling_loop(self):
        # 1. 构建异构图
        gdata = build_hetero_graph(nodes, edges, self.ships, self.tasks, ports)

        # 2. HGT 编码
        h_r, h_s, h_t, attn = self.encoder(gdata)

        # 3. 扫描事件
        deviations = compute_deviations(self.ships, self.assignments, self.step)
        trigger_prob, scope_probs = self.gate(deviations)

        if trigger_prob > 0.5:
            # 触发重分配
            scope = argmax(scope_probs)
            self.execute_reallocation(scope, h_s, h_t)
        elif self.has_unassigned_tasks():
            # 正常分配: 为新任务分配合适的船
            self.execute_normal_assignment(h_s, h_t)

        # 4. 发布分配结果
        self.publish_assignments()

    def execute_normal_assignment(self, h_s, h_t):
        ship_loads = compute_current_loads(self.ships)
        assigned_mask = compute_assigned_mask(self.tasks)

        for _ in range(len(self.tasks)):

            _, ship_idx, task_idx, _ = self.assigner.act(
                logits, ship_loads, self.ships, self.tasks, assigned_mask,
```

### 6.4 与 Navigation2 栈的集成

TAMP 调度器输出的是任务序列（装货→航行→卸货），底层路径规划由 Nav2 处理：

```
TAMP 输出: Ship_i 执行 [T5, T2, T8]
    ↓
转换为行为树:  NavigateTo(P5) → Load → NavigateTo(D5) → NavigateTo(P2) → Load → ...
    ↓
Nav2 处理:    全局规划器 → 局部规划器 → 控制器
```

## 8. 部署建议

### 8.1 离线训练 → 在线推理

```
离线阶段 (GPU服务器):
  1. 准备历史航行数据 / 仿真数据
  2. 运行 main.py 完成 Phase 1→2→3 训练
  3. 导出 model.pt (encoder + assigner + pointer + gate)

在线阶段 (ROS边缘设备):
  1. 加载 model.pt (仅推理，无需梯度)
  2. 运行 TAMPScheduler 节点
  3. 每个调度周期 (~1s) 执行一次推理
  4. PPO/Gate 推理耗时 ~10ms (CPU), 可实时运行
```

### 8.2 模型更新策略

- 定期（如每周）用新收集的数据重新训练 Gate
- 重大航线变更时重新训练编码器和分配器
- 线上只做推理，不做在线学习（避免灾难性遗忘）

### 8.3 性能预估

| 规模 | 图构建 | HGT 编码 | 分配推理 | 总延迟 |
|------|--------|----------|----------|--------|
| 5船×10任务, 100路网节点 | <1ms | <1ms | <1ms | ~5ms |
| 30船×60任务, 400路网节点 | ~20ms | ~10ms | ~30ms | ~60ms |
| 100船×300任务, 1600路网节点 | ~100ms | ~50ms | ~200ms | ~350ms |

（测试环境：Intel i7 CPU, PyTorch 2.x, 无 GPU）
