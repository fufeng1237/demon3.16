# 代码修改计划 — 基于系统最终方案

## 核心变化

```
旧: ALNS 优化 Task 分配 → 输出 Task ID 序列
新: ALNS 优化 RoadNode 访问序列 → 输出 (node_id, action) 序列
    action ∈ {PICKUP(task_id), DELIVERY(task_id)}

旧输出: Ship_0: [T1, T3, T5]
新输出: Ship_0: [Node15, Node25(PICKUP T3), Node42(PICKUP T5), Node87(DELIVERY T3), Node96(DELIVERY T5)]
```

---

## 一、Route 数据结构

```python
@dataclass
class RouteNode:
    node_id: int           # RoadNode ID
    action: str            # "PICKUP" | "DELIVERY" | "PASS"
    task_id: int = -1      # 关联的任务 (PASS 时为 -1)

# Route = List[RouteNode]
# 例: [RouteNode(15, "PASS"), RouteNode(25, "PICKUP", 3), RouteNode(87, "DELIVERY", 3)]
```

## 二、需要修改的文件

### 2.1 `graph_evaluator.py` — 小改

```
改动: InsertCost 计算从 task 序列改为 RouteNode 序列

新增方法:
  evaluate_route_insert(ship, route_nodes, task) → (best_pos, insert_cost)
    - 在 RouteNode 序列中找插入一对 PICKUP/DELIVERY 节点的最佳位置
    - 考虑: 先装后卸, 容量约束, 路径最短
```

### 2.2 `alns_scheduler.py` — 重写核心

```
改动: 优化对象从 Task 列表改为 RouteNode 列表

重写方法:
  build_initial_routes()     → 输出 {ship_id: List[RouteNode]}
  _route_cost()              → 计算 RouteNode 序列总航程
  _best_insert()             → 在 RouteNode 序列中插入 PICKUP+DELIVERY 对
  _validate_route()          → 验证 RouteNode 序列约束
  _apply_destroy()           → 移除 PICKUP+DELIVERY 对
  _apply_repair()            → 插入 PICKUP+DELIVERY 对

新增:
  _route_to_node_list()      → RouteNode 序列 → 纯节点 ID 列表 (输出用)
  _node_list_with_actions()  → 带标注的节点序列 (显示用)
```

### 2.3 新增 `route_executor.py`

```
职责: 按 RouteNode 序列执行运输

class RouteExecutor:
    execute_step(ship, dt)   → 推进船沿 Route 航行
    advance_to_next_node()   → 船到达下一个 RouteNode
    handle_pickup()          → 装货
    handle_delivery()        → 卸货
    update_ship_state()      → 更新船状态
    update_task_state()      → 更新任务状态
```

### 2.4 新增 `rolling_horizon.py`

```
职责: 周期性重优化未冻结的 Route 部分

class RollingHorizon:
    optimize(ship, frozen_count=2)  → 优化冻结点之后的 RouteNode
    触发: 每完成一个 RouteNode 时
```

### 2.5 新增 `route_repair.py`

```
职责: 异常时局部修复 Route

class RouteRepair:
    repair(event, ships, routes) → 释放受影响 RouteNode → ALNS 局部重优化
```

### 2.6 新增 `event_detector.py`

```
职责: 检测异常事件

class EventDetector:
    detect(ships, tasks) → List[Event]
```

### 2.7 新增 `scheduler.py`

```
职责: 整合所有模块的主调度器

class Scheduler:
    initialize()     → 构建 Graph + ALNS 初始 Route
    step()           → 执行一个时间步
    run()            → 主循环
```

## 三、ALNS 核心逻辑变化

### 旧 (Task 序列)

```python
# 旧: 优化 Task ID 列表
route = [T1, T3, T5]     # Task ID 序列

def _route_cost(ship, task_ids):
    cur = ship.current_node
    for tid in task_ids:
        task = tasks[tid]
        total += dist[cur, task.pickup_node]
        total += dist[task.pickup_node, task.delivery_node]
        cur = task.delivery_node

def insert_task(ship, task, pos):
    # 插入一个 task_id 到位置 pos
    new_seq = seq[:pos] + [tid] + seq[pos:]
```

### 新 (RouteNode 序列)

```python
# 新: 优化 RouteNode 序列
route = [
    RouteNode(15, "PASS"),
    RouteNode(25, "PICKUP", task_id=3),
    RouteNode(87, "DELIVERY", task_id=3),
    RouteNode(42, "PICKUP", task_id=5),
    RouteNode(96, "DELIVERY", task_id=5),
]

def _route_cost(ship, route_nodes):
    cur = ship.current_node
    for rn in route_nodes:
        total += dist[cur, rn.node_id]
        cur = rn.node_id

def insert_task(ship, task, pos_pickup, pos_delivery):
    # 插入一对: PICKUP + DELIVERY 两个 RouteNode
    rn_pickup = RouteNode(task.pickup_node, "PICKUP", task.task_id)
    rn_delivery = RouteNode(task.delivery_node, "DELIVERY", task.task_id)
    new_seq = seq[:pos_pickup] + [rn_pickup] + seq[pos_pickup:pos_delivery] + [rn_delivery] + seq[pos_delivery:]
```
