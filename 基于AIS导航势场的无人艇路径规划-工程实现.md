# 基于AIS导航势场的无人艇路径规划——工程实现与改进

## 一、与原论文的核心差异

原论文（`基于AIS导航势场的无人艇路径规划.md`）提出了方法框架，但在工程落地过程中，三个关键假设在实际数据上不成立，需要实质性修改：

| 原论文假设 | 工程现实 | 本方案改进 |
|------------|----------|------------|
| 逐点COG可正确分类顺/逆流 | COG瞬时值含噪声，单船航向波动导致分类跳变 | **按轨迹级别判定方向**：每条船根据起止经度位移统一归入东行或西行集 |
| 全局标量方向参数 $d$ 足以描述航道 | 分岔口水域不同通道有不同流向，$d$ 只有一个值 | **栅格级矢量方向场 $\mathbf{d}(\mathbf{p})$**：每个栅格独立计算期望航向单位矢量 |
| "只惩罚走错"即可引导路径 | 纯惩罚没有中心吸附，路径可在正确航道内任意游走，不一定走最优带 | **拆分方向约束与位置质量**：方向项严格惩罚逆行；位置质量项温和偏好高密度中心 |

---

## 二、导航势场构建

### 2.1 轨迹级方向分离（关键改进 1）

**原论文方法**：对每个 AIS 轨迹点，按对地航向 (COG) 与全局参考方向的夹角 $< 90^\circ$ 归入顺流集、否则逆流集。

**问题**：(1) COG 瞬时值含观测噪声；(2) 同一艘船的不同点可能被归入不同集合，破坏轨迹一致性；(3) 全局参考方向 $\theta_{\text{ref}}$ 本质是一个无物理含义的魔法数字。

**本方案方法**：以**每条船为单位**判定方向。每个 XLS 文件即一艘船的完整航次，取前 $\frac{1}{3}$ 与后 $\frac{1}{3}$ 轨迹点的经度中位数计算净位移：

$$
\text{方向} = \begin{cases} \text{东行（顺流）}, & \text{median}(lon_{\text{后}}) - \text{median}(lon_{\text{前}}) > 0 \\ \text{西行（逆流）}, & \text{otherwise} \end{cases}
$$

整条轨迹的所有点统一归入同一集合，保证了轨迹在时间维度上的方向一致性。**不再依赖 COG 或任何全局参考角度**。

最终数据：**东行轨迹 47 条 (2,775 点) 构成顺流集 $\mathcal{T}_{\downarrow}$，西行轨迹 88 条 (5,519 点) 构成逆流集 $\mathcal{T}_{\uparrow}$**。

### 2.2 分向核密度估计

沿用论文公式 (1)，对两集合分别做高斯核密度估计：

$$
f_{\downarrow}(\mathbf{p}) = \sum_{\mathbf{p}_i \in \mathcal{T}_{\downarrow}} \exp\!\left[-\frac{\lVert\mathbf{p} - \mathbf{p}_i\rVert^2}{2h^2}\right], \qquad
f_{\uparrow}(\mathbf{p}) = \sum_{\mathbf{p}_i \in \mathcal{T}_{\uparrow}} \exp\!\left[-\frac{\lVert\mathbf{p} - \mathbf{p}_i\rVert^2}{2h^2}\right]
$$

核带宽 $h = 2.0\text{m}$（较原论文更小，减少不同方向信号的空间重叠）。栅格分辨率 $0.5\text{m}$，网格尺寸 $78 \times 152$，覆盖 $76\text{m} \times 39\text{m}$ 水域。

### 2.3 对比度拉伸

在归一化后施加幂次变换以强化方向区分度：

$$
\bar{\Phi}'(\mathbf{p}) = \text{sgn}\big(\bar{\Phi}(\mathbf{p})\big) \cdot \big|\bar{\Phi}(\mathbf{p})\big|^\gamma, \quad \gamma = 0.6
$$

效果：$|\bar{\Phi}| = 0.1 \to 0.25$，$|\bar{\Phi}| = 0.5 \to 0.66$，极值不变。混合区（$|\bar{\Phi}| \le 0.1$）面积从 3,008 栅格降至 51 栅格。

### 2.4 期望航向矢量场（关键改进 2）

**原论文方法**：用全局标量 $d \in \{+1, -1\}$ 表示期望航向，依赖顶层图搜索确定航段方向。分岔口不同通道的方向差异无法用单一标量表达。

**本方案方法**：构建**逐栅格的单位矢量方向场** $\mathbf{d}(\mathbf{p}) \in \mathbb{R}^2, \lVert\mathbf{d}\rVert = 1$。计算方式为 KDE 加权平均：

$$
d_x(\mathbf{p}) = \frac{\sum_i \sin(\text{COG}_i) \cdot K(\mathbf{p}, \mathbf{p}_i)}{\sum_i K(\mathbf{p}, \mathbf{p}_i)}, \quad
d_y(\mathbf{p}) = \frac{\sum_i \cos(\text{COG}_i) \cdot K(\mathbf{p}, \mathbf{p}_i)}{\sum_i K(\mathbf{p}, \mathbf{p}_i)}
$$

然后归一化。**COG 的 $0^\circ$ 为北（$\uparrow$），$\sin/\cos$ 的 $0^\circ$ 为东（$\rightarrow$），需做坐标转换：$d_x = \sin(\text{COG})$，$d_y = \cos(\text{COG})$——这是工程实现中踩过的坑。**

$\mathbf{d}(\mathbf{p})$ 的物理含义：在该位置处，历史船舶的平均行驶方向。分岔口的上通道和下通道会有不同的 $\mathbf{d}(\mathbf{p})$，A\* 在不同区域自动收到不同的方向引导。

## 三、航道引导 A\* 代价函数（关键改进 3）

### 3.1 拆分方向约束与位置质量

**原论文方法**：状态 $c_{\text{lane}} = \max(0, -d \cdot \bar{\Phi})$。正确航道代价恒为 0，但无中心吸附梯度；错误航道代价随偏离程度递增。

**问题**：纯惩罚意味着在正确航道内部，所有位置代价相同（均为 0），A\* 没有动力走向密度最高的航道中心脊线——它可能贴着正确航道的边缘走，而边缘恰好紧邻障碍物或浅滩。

**本方案方法**：将代价拆分为两项：

$$
c_{\text{lane}}(\mathbf{p}_i, \mathbf{p}_j) = \underbrace{\lambda_{\text{dir}} \cdot \max(0, -\mathbf{v} \cdot \mathbf{d}) \cdot |\bar{\Phi}|^2}_{\text{方向约束（强）}} + \underbrace{\lambda_{\text{qual}} \cdot (1 - |\bar{\Phi}|) \cdot \lVert\mathbf{p}_j - \mathbf{p}_i\rVert}_{\text{位置质量（弱）}}
$$

其中 $\mathbf{v} = \frac{\mathbf{p}_j - \mathbf{p}_i}{\lVert\mathbf{p}_j - \mathbf{p}_i\rVert}$ 为步进方向的单位矢量。

**方向约束项**：仅当 USV 走反方向（$\mathbf{v} \cdot \mathbf{d} < 0$）时产生惩罚，强度与势场强度 $|\bar{\Phi}|^2$ 成正比。正确方向或横穿时均为 0。

**位置质量项**：当 USV 处于低密度区域（$|\bar{\Phi}|$ 小）时，付出与步长成正比的小额代价，引导路径偏好高密度航道中心。$\lambda_{\text{qual}} = 1.0$ 约为 $\frac{1}{400}$ 的方向惩罚强度。

### 3.2 A\* 步进代价

$$
c(\mathbf{p}_i, \mathbf{p}_j) = \lVert\mathbf{p}_j - \mathbf{p}_i\rVert + c_{\text{lane}}(\mathbf{p}_i, \mathbf{p}_j)
$$

A\* 总评价函数（引导模式 $w_h = 0.1$，传统模式 $w_h = 1.0$）：

$$
f(n) = g(n) + w_h \cdot \lVert\mathbf{p}_n - \mathbf{p}_{\text{goal}}\rVert
$$

### 3.3 代价行为表 ($\lambda_{\text{dir}} = 5.0$, $\lambda_{\text{qual}} = 1.0$)

| 场景 | $|\bar{\Phi}|$ | $\mathbf{v} \cdot \mathbf{d}$ | 方向项 | 质量项 | 总代价 |
|------|:---:|:---:|--------|--------|--------|
| 航道中心，正确方向 | 1.0 | +1 | 0 | 0 | **0** |
| 航道边缘，正确方向 | 0.3 | +1 | 0 | 0.35 | **0.35** |
| 航道中心，逆行 | 1.0 | -1 | 5.0 | 0 | **5.0** |
| 航道边缘，逆行 | 0.3 | -1 | 0.45 | 0.35 | **0.80** |
| 自由水域 | $\approx 0$ | 任意 | 0 | $\approx$ 步长 | **≈ 纯距离** |
| 分岔口，正确方向 | 0.5 | +1 | 0 | 0.25 | **0.25** |

## 四、与原论文的差异总结

### 4.1 为什么要改

| 原论文设计 | 工程问题 | 本方案改进 |
|------------|----------|------------|
| 分层规划：顶层图搜索 + 中层分段 A\* | 当前尚未实现顶层，无法自动确定 $d_k$ | **底层先自立**：用矢量场 $\mathbf{d}(\mathbf{p})$ 替代标量 $d$，每个栅格自带方向判断，暂时不用顶层 |
| 顺/逆流按 COG 与 $\theta_{\text{ref}}$ 夹角分类 | (a) COG 有噪声；(b) $\theta_{\text{ref}}$ 是魔法数字；(c) 同船不同点可能被分到不同集合 | **轨迹级方向判定**：每船按起止位移归入东行或西行，整船一致，无魔法数字 |
| 代价 $c_{\text{lane}} = \max(0, -d \cdot \bar{\Phi})$ | 纯惩罚无中心吸附；分岔口 $d$ 不适用 | **拆分方向约束+位置质量**：方向项用矢量点积替代标量符号判断；位置质量项温和吸附航道中心 |
| 标量方向参数 $d \in \{+1,-1\}$ | 单值无法表达不同通道的不同流向 | **矢量场 $\mathbf{d}(\mathbf{p})$**：逐栅格独立方向，分岔口自动切换 |
| 核带宽 $h$ 固定 | 不区分数据稀疏密度 | $h=2.0\text{m}$ 更小，减少方向信号重叠；对比度拉伸 $\gamma=0.6$ 强化区分 |

### 4.2 当前架构

```
                    ┌─────────────────────────────┐
                    │   离线 Python 脚本            │
                    │                              │
                    │  AIS XLS → 轨迹方向判定       │
                    │         → 分向 KDE → Φ̄ 场    │
                    │         → 矢量场 d(p)        │
                    └──────────┬──────────────────┘
                               │ CSV 文件
                               ▼
┌──────────────────────────────────────────────────────────┐
│                在线 ROS C++ 节点                          │
│                                                          │
│  AIS Planner Node                                        │
│  ├─ 加载 Φ̄ + d(p) 矢量场                                  │
│  ├─ A* 搜索: stepCost = dist + c_lane                    │
│  │   c_lane = λ_dir × max(0,-v·d) × |Φ̄|²                │
│  │          + λ_qual × (1 - |Φ̄|) × dist                  │
│  └─ 输出: /ais_planned_path, /ais_traditional_path       │
│                                                          │
│  AIS Path Optimizer Node                                 │
│  └─ 三目标梯度下降平滑: 数据保真+几何平滑+航道偏离        │
│     输出: /ais_smoothed_path                             │
└──────────────────────────────────────────────────────────┘
```

### 4.3 优点

1. **无魔法数字**：不再需要 $\theta_{\text{ref}}$ 或任何全局参考方向参数。轨迹方向来自数据本身，矢量场自动给出每点的最优航向。

2. **分岔口原生处理**：上通道和下通道的 $\mathbf{d}(\mathbf{p})$ 自动指向不同方向，A\* 在不同区域收到正确的引导，无需顶层图搜索介入。

3. **方向与位置解耦**：方向走反 → 强力惩罚；方向正确但位置偏 → 温和偏好。路径不会在正确航道内被不必要地挤到中心线，也不会在逆行时轻易逃脱。

4. **对比度强化**：$\gamma = 0.6$ 的幂次变换将弱信号放大，使顺/逆流区域泾渭分明。

5. **C++ 侧简洁**：只加载 3 个 CSV（$\bar{\Phi}$、$d_x$、$d_y$），步进代价计算只需一次 GridMap 查表 + 一次向量点积。

### 4.4 尚未实现（后续工作）

- **顶层河道网络图搜索**：多航段自动划分与方向切换
- **对数平滑 / 独立极值归一化**：当数据极端不平衡时（不是当前场景）可作为备选
- **异构核带宽**：稀疏侧用更大 $h$ 保证连通性，需仔细调参避免信号漫溢
